📌 프로그램 시연 운영 매뉴얼 (동기부여 목적)
- 학생들에게 동기부여를 위해 프로그램을 시연합니다. (한 수업당 1~2개 시연)
- 시연 타이밍은 쉬는 시간 직후 또는 집중도가 떨어지는 시점에 진행하는 것을 권장합니다!

1. 단순 관람이 아닌 ‘참여’로 이어지도록 합니다
- 학생들이 단순히 보는 것에서 끝나지 않도록,
  희망하는 학생은 직접 체험할 수 있도록 유도합니다.
- 필요 시, 선생님 자리 또는 별도 기기에서 짧게라도 직접 실행 경험을 제공해 주세요.

2. 수업 흐름과 연결되도록 설명합니다
- 단순히 “멋있는 결과물”로 끝나지 않도록, 현재 배우는 개념과 연결해서 설명해 주세요.
  예: “지금 배우는 조건문/반복문이 이런 움직임에 사용된다.”
- 학생 스스로가 “내가 배우는 게 여기에 쓰이는구나”를 이해하도록 해주세요.

3. 사전 준비 (필수)
- 시연 전, 담당 선생님은 필요한 모듈 및 실행 환경을 반드시 사전 점검해 주세요.
- 오류 발생 시 수업 흐름이 끊기지 않도록 미리 실행 테스트를 권장합니다.
- 저장소의 모든 코드는 Mac 환경 기준으로 작성되어 있으며,
  운영체제에 따라 일부 코드가 다르게 동작할 수 있으니 참고 부탁드립니다.

4. 동기부여 멘트 가이드
- 프로그램을 보여줄 때는 아래와 같은 메시지를 함께 전달해 주세요:
  “다른 반 학생이 직접 만든 작품이다.”
  “지금 배우는 과정을 잘 끝내면 OO이도 만들 수 있다.”
👉 핵심은 [나도 할 수 있다]는 인식 형성입니다.
